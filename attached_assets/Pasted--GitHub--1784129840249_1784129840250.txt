اعمل داخل مشروع «مدار» الحالي المستورد من GitHub، ولا تنشئ مشروعًا جديدًا ولا تحذف أو تغيّر الصفحات الحالية. افحص جميع الملفات أولًا، ثم أنشئ لوحة تحكم متكاملة للمعلمة مستوحاة من الصورة المرفقة ومتوافقة بصريًا مع ألوان وهوية موقع مدار.

المطلوب:

1. التصميم العام:

* واجهة عربية كاملة باتجاه RTL.
* تصميم احترافي وهادئ ومتجاوب مع الكمبيوتر والجوال.
* قائمة جانبية مرتبة، وترويسة تحتوي اسم المعلمة وصورتها والتنبيهات وزر تسجيل الخروج.
* استخدم خط Tajawal وألوان موقع مدار الحالية.
* اجعل الأزرار متوسطة أو صغيرة ولا تغطي الشاشة.
* أضف حركات انتقال بسيطة وإضاءة أنيقة عند مرور الماوس.
* استخدم أيقونات احترافية ومتناسقة.

2. الصفحة الرئيسية للوحة التحكم:

* بطاقات تعرض عدد الطالبات، الاختبارات المنشورة، الاختبارات المكتملة، متوسط الفصل، والطالبات المحتاجات إلى دعم.
* مخطط مختصر لمستوى الفصل.
* آخر الأنشطة والتنبيهات.
* اختصارات لإضافة اختبار أو طالبة أو ملاحظة.

3. قسم «معلوماتي»:

* عرض وتعديل اسم المعلمة.
* عرض وتعديل البريد الإلكتروني.
* تغيير كلمة المرور من خلال: كلمة المرور الحالية، الجديدة، وتأكيد الجديدة.
* التحقق من صحة البيانات وإظهار رسائل نجاح أو خطأ واضحة.
* عدم تخزين كلمات المرور داخل JavaScript أو المتصفح، واستخدام طريقة آمنة ومشفّرة.

4. قسم «ملف المتابعة»:

* ملف متابعة لكل طالبة يشمل الدرجات، مستوى المهارات، الملاحظات، الحضور، الواجبات، ونسبة التقدم.
* البحث عن طالبة وتصفيتها حسب الفصل أو المستوى.
* إضافة وتعديل الملاحظات.
* طباعة التقرير وتصديره إلى PDF أو Excel إن أمكن بطريقة مجانية.

5. قسم «الطالبات» ويحتوي خانتين:

* «أسماء الطالبات»: جدول منظم يعرض اسم الطالبة، بريدها الإلكتروني، الفصل، مستوى التقدم، وآخر نشاط.
* «إدارة الطالبات»: إضافة طالبة، تعديل معلوماتها، حذفها بعد التأكيد، واستيراد قائمة طالبات من CSV.
* أضف البحث والفرز والتصفية والتنقل بين الصفحات.
* عند الضغط على اسم الطالبة افتح ملفها الكامل.

6. قسم «الاختبارات» ويحتوي ثلاث خانات:

* تشخيصي قبلي.
* تشخيصي بعدي.
* اختبارات قصيرة.

داخل كل خانة أريد:

* إنشاء اختبار جديد وتعديله وحذفه ونسخه.
* إضافة أسئلة اختيار من متعدد وصح أو خطأ وإجابة قصيرة.
* تحديد المهارة والدرجة ووقت الاختبار وتاريخ البداية والنهاية.
* حفظ الاختبار كمسودة أو نشره.
* عرض حالة الاختبار وعدد الطالبات اللاتي أجبن.
* تصحيح تلقائي للأسئلة الموضوعية وعرض النتائج.

7. قسم «تحليل النتائج» ويحتوي ثلاث خانات:

* «تحليل لكل طالبة»: درجات الطالبة، تطورها، المهارات المتقنة والمهارات التي تحتاج إلى دعم، مع رسوم بيانية واضحة.
* «تحليل الفصل العام»: متوسط الفصل، أعلى وأقل درجة، توزيع الدرجات، نسبة النجاح، ومقارنة التشخيص القبلي والبعدي.
* «تحليل كل مهارة»: نسبة إتقان كل مهارة، الطالبات المتقنات، والطالبات المحتاجات إلى تدخل علاجي.
* أضف فلاتر حسب الفصل والاختبار والفترة الزمنية.
* استخدم رسومًا بيانية احترافية ومتجاوبة.

8. أكمل لوحة التحكم بأقسام مفيدة مثل:

* التقارير.
* الإشعارات.
* الفصول والمجموعات.
* الإعدادات.
* سجل الأنشطة.
* تسجيل الخروج.

المتطلبات البرمجية:

* اجعل جميع الأزرار والصفحات والعمليات تعمل فعليًا، وليس مجرد واجهات أو رسائل تجريبية.
* أنشئ نظام تسجيل دخول وصلاحية «معلمة».
* استخدم قاعدة بيانات حقيقية وآمنة لحفظ المعلمات والطالبات والاختبارات والنتائج والمهارات والملاحظات.
* أنشئ بيانات تجريبية واضحة لعدد 12 طالبة وعدة اختبارات ومهارات حتى أستطيع تجربة اللوحة.
* استخدم تحققًا من البيانات في الواجهة والخادم، ولا تضع كلمات مرور أو مفاتيح سرية داخل ملفات الواجهة.
* أضف حالات التحميل والفراغ والأخطاء ورسائل التأكيد قبل الحذف.
* حافظ على ملفات وصفحات الموقع الحالية واربط لوحة المعلمة بها بطريقة سليمة.
* أضف زرًا مناسبًا للدخول إلى لوحة المعلمة، لكن لا تعرض بيانات المعلمة أو الطالبات دون تسجيل الدخول.
* اختبر جميع الصفحات والروابط والأزرار على الكمبيوتر والجوال.
* أصلح أي خطأ يظهر في Console.
* استخدم خدمات مجانية قدر الإمكان. لا تشترك في أي خدمة مدفوعة ولا تنشر الموقع ولا تدفع التعديلات إلى GitHub قبل موافقتي.
* إذا احتجت إلى تفعيل قاعدة بيانات أو تسجيل دخول أو إدخال مفتاح خارجي، توقّف واشرح لي المطلوب والتكلفة أولًا.

ابدأ بفحص المشروع، ثم اعرض لي خطة قصيرة وهيكل الصفحات وقاعدة البيانات. بعد ذلك نفّذ المشروع على مراحل، واعرض المعاينة بعد كل مرحلة. لا تغيّر المستودع على GitHub ولا تنشر النسخة النهائية إلا بعد أن أوافق صراحة.
